# Web Portal & Media Sorter

Это комплексное решение для управления медиафайлами на NAS (TrueNAS) с помощью веб-интерфейса, мобильного приложения и локального агента под Windows.

## Архитектура системы

Проект состоит из нескольких взаимосвязанных компонентов:

1. **Backend (FastAPI)** (`/backend`):
   Обеспечивает REST API для взаимодействия с файловой системой (SMB-шары), базой данных (SQLite) и клиентами. Включает логику сортировки медиа (по папкам `/year/month/day`), обработки изображений и кэширования превьюшек. Запускается на облачном сервере Debian.

2. **Frontend (React + Vite)** (`/frontend` или `frontend_dist`):
   Веб-интерфейс (SPA) для просмотра альбомов, управления фотографиями и мониторинга системы. Общается напрямую с FastAPI бэкендом. 

3. **Android App (React Native)** (`/android`):
   Нативное мобильное приложение с похожим на веб-интерфейс функционалом (галерея, свайпы, управление медиафайлами). Собирается как APK и скачивается с бэкенда.

4. **Windows Agent** (`/agent`):
   Служба (System Tray приложение) для локального ПК, находящегося в одной сети с TrueNAS. 
   - Пробрасывает SSH реверс-туннели к облачному Debian-серверу (порт 445 для SMB и 8001 для локальных метрик).
   - Транслирует системные метрики ПК и TrueNAS (запрашивает их по API TrueNAS).

## Инструкции по развертыванию

### Настройка Backend (Debian)
1. Установите зависимости Python: `pip install -r requirements.txt`.
2. Настройте Nginx на проксирование к FastAPI.
3. Запустите бэкенд: `uvicorn main:app --host 0.0.0.0 --port 8000`.

### Сборка Frontend
1. Перейдите в папку `frontend_src`.
2. Выполните: `npm install` и `npm run build`.
3. Собранные статические файлы будут в `dist/`, бэкенд раздает их по корневому пути `/`.

### Сборка Windows Agent
Для сборки `WebPortalAccessAgent.exe` вам потребуется PyInstaller и Inno Setup.
1. Откройте консоль в `agent/`.
2. Выполните сборку: 
   ```bash
   pyinstaller --onefile --noconsole --name WebPortalAccessAgent --icon=icon.ico --version-file=version.txt windows_agent.py
   ```
3. Откройте `agent_installer.iss` с помощью Inno Setup (или скомпилируйте через консоль: `iscc agent_installer.iss`). Это создаст полноценный инсталлятор в `dist/`.

## Структура репозитория
* `/agent/` - Исходный код и скрипты сборки Windows-агента.
* `/backend/` - Серверный код (FastAPI). *(Примечание: файлы могут находиться в корне в процессе рефакторинга)*
* `/scripts/` - Полезные утилиты для деплоя и сборки на PowerShell.
* `/archive/` - Одноразовые скрипты и эксперименты, сохраненные для истории.
